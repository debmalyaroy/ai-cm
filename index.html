1. Why AI is Required

Traditional rules engines require anticipating every input variance. AI-CM uses Generative AI to navigate the context-heavy world of retail analytics seamlessly.

💬

The "Natural Language" Problem

Rule-based NLP fails on novel phrasing and typos. AI understands semantic weight, allowing Category Managers to ask questions in plain business English.

📈

The "SQL Generation" Problem

Standard BI breaks on queries requiring novel JOINs. AI-CM builds SQL dynamically and self-heals based on error feedback—no manual templates needed.

Workflow ROI

AI-CM flips the 70/30 time waste ratio, automating data retrieval layer to free up managers for strategic planning.

2. The Cognitive Workforce

🎛️

Supervisor

Orchestrator

Technical Architecture

Few-shot intent classifier with a regex-based Clarification Gate to handle vague queries at zero token cost.

AI Justification

Isolates intent in multi-context strings where keyword matchers trigger false positives.

📊

Analyst

Text-to-SQL

Technical Architecture

Uses ReAct pattern (Reason + Act). If SQL fails, the agent interprets the error message and self-heals syntax dynamically.

AI Justification

Autonomous navigation of 15+ relational tables eliminates the maintenance of brittle SQL templates.

🧠

Strategist

Chain-of-Thought

Technical Architecture

Integrated with pgvector Semantic Index. Injects relevant historical context into the prompt via cosine similarity.

AI Justification

Evaluates complex causal chains (e.g. correlating competitor price drops with stock levels) that rules can't model.

📋

Planner

Action Engine

Technical Architecture

Outputs strict JSON conforming to action_log schema for direct persistence and UI rendering.

AI Justification

Maps fuzzy business intentions to concrete execution parameters with contextual confidence scores.

✉️

Liaison

Communications

Technical Architecture

Context-aware drafting loop. Uses RAG-retrieved tone-of-voice and supplier history to personalize prose.

AI Justification

Replaces robotic boilerplate with tactful communication, saving the user significant manual drafting time.

🐕

Watchdog

Monitoring

Technical Architecture

Operates on a Distributed Cron Scheduler using DB locks. Hybrid SQL/LLM logic for cost-efficiency.

AI Justification

Provides "Reasonable Checks" on data anomalies to ensure managers only receive high-quality alerts.

⚖️

Recommender

Rule Engine

Technical Architecture

Pure Go implementation. Uses pre-compiled SQL queries for immediate response for known inventory/price thresholds.

Architecture Role

Supports the AI layer with zero-latency baseline heuristic data.
